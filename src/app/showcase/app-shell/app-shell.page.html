App Shell

After analyzing multiple use cases, we realized most UIs can be deconstructed into two data bindable element primitives: text and images.

These Shell Elements are the building blocks to display loading states.

Shell Elements Aspect Ratio Image Shell Text Shell

You can combine the different shell elements in multiple ways to fit your needs

Text on top of a background-image container

By following the reactive programming paradigm to handle asynchronous data streams, we can use Observables to manage the different states (loading / loaded) of the data binded to our shell elements.

Note: If you are new to reactive programming in Javascript, I strongly recommend you to read this guide.

Simple Data Binding Data Binding using plain Observables

You can go a step further and use our DataStore class which provides extra functionalities to plain Observables.

The main responsability of the DataStore is to store the data stream in a timeline property and append shell values to the Observable data sources through the AppendShell static method.
