DataStore - Dependant Data Sources

This example kinda mixes the two previous examples (multiple data stores, one depending on the other).

The uses cases in which to use this approach would be if you have different data sources that depend on each other, but you want the different sections of the view to 're-load' independently one from the other.

This contrasts the use case of the Combined Data Source where you transform multiple data sources into one, causing the hole view to be 're-loaded' as it's binded to just one combined Observable.

A good example can be a post and its comments. The comments depend on the post but you want them to 're-load' independently from the post.
