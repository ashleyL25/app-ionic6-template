Text Shell

This component is useful when you want to show a loading indicator while fetching text data from the server.

The <app-text-shell> basically works by wrapping the text node with a loading indicator while you are fetching data.

While there are empty values the component adds some 'loading' styles and animations. Whereas while there are non empty values, the loading state is removed.

Try toggling the loading state of the text shells below by changing the value of this textarea.

You can reset the 'loading' state by clearing the textarea value.

Prepend and Append text to the data binded

There are cases where we need to add some text before or after the value binded to the <app-text-shell>. For example if we are loading a price value and we want to put the '$' before the price. 

  <app-text-shell [data]="(sampleTextShellData) ? '$'.concat(' ', sampleTextShellData) : null"></app-text-shell>
  
<app-text-shell [data]="(sampleTextShellData) ? sampleTextShellData.concat(' years old') : null"></app-text-shell>

Other use cases

The <app-text-shell> element can be used alone or wrapped with a text tag (h1, h2, h3, p, etc).

In case you wrap it with a heading element, we included basic styles (src/theme/shell-defaults.scss) to match the different heading styles (mainly line-height associated to each heading). 

:root {
  h1 > app-text-shell {
    --text-shell-line-height: 32px;
    --text-shell-line-color: #CCC;
  }
}

By matching the line-height of the text-shell with the line-height of the wrapper element, we avoid page height bumps after the data gets loaded into the shells. 

<h1>
  <app-text-shell [data]=""></app-text-shell>
</h1>

<h2>
  <app-text-shell [data]=""></app-text-shell>
</h2>

You can add/remove the following Attributes to adjust the shell element behavior

Multi-line support

You can set up-to 6 lines 

<p>
  <app-text-shell lines="6" [data]=""></app-text-shell>
</p>

Set different animation options

You can choose between no animation (default), bouncing animation, and gradient background with line masks on top 

<p>
  <app-text-shell animation="bouncing" lines="2" [data]=""></app-text-shell>
</p>


<p>
  <app-text-shell animation="gradient" lines="2" [data]=""></app-text-shell>
</p>

app-text-shell {
  --text-shell-background: #FFF;
  --text-shell-line-color: transparent;
}

In this mode, the --text-shell-background property refers to the masks colors.

Note: When using a gradient animation, make sure you set the line-color to transparent so the background animation that's beneath the masks can be seen.

You can also override these CSS 4 variables to adjust the shell element style

Line color
app-text-shell {
  --text-shell-line-color: rgba(233, 30, 99, .25);
}

Background

Background by default is set to transparent. 

app-text-shell {
  --text-shell-line-color: #FFFFFF;
  --text-shell-background: #000000;
}

Background Animation

Note: These properties are only applied when using animation="gradient". 

app-text-shell {
  --text-shell-line-color: transparent;
  --text-shell-background: #000000;
  --text-shell-animation-background: rgba(255, 3, 109, 0.6);
  --text-shell-animation-color: rgba(156, 4, 68, 0.7);
}

Line height
app-text-shell {
  --text-shell-line-height: 40px;
}

Line gutter
app-text-shell {
  --text-shell-line-gutter: 10px;
}

Max-width

By default, the <app-text-shell> element will fill it's container width. You can adjust this by setting a temporary max-width while the shell loading state is present. 

.sample-text-max-width > app-text-shell {
  max-width: 50%;

  &.text-loaded {
    max-width: unset;
  }
}
Min-width

When the <app-text-shell> parent container doesn't have a width defined, it may happen that the <app-text-shell> inherits an undefined width and thus it's not shown.

In these cases setting a temporary min-width fixes the issue.

Note: If the parent doesn't have a defined width, it will fail to set a min-width based on a percentage of it's parents width. You will have to use absolute units (px, ch, etc). 

.sample-text-min-width > app-text-shell {
  min-width: 80px;

  &.text-loaded {
    min-width: 0px;
  }
}
